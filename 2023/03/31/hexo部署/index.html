hexo + github部署

hexo安装

  • git安装
  • nodejs安装
  • hexo安装
    • 使用npm安装hexo npm install -g hexo-cli
    • 初始化博客 hexo init blog
    • 生成 hexo g= hexo generate
    • 本地预览 hexo s=hexo server
    • 打开localhost:4000预览

部署到github

  • github新建repo

    注意repo名为github用户名.github.io

    例如 用户名为hybai的用户 新建repo为hybai.github.io
  • 将hexo和github链接

    找到创建的博客文件夹下的_config.yml文件

    修改deploy
    deploy:
type: git
repo: git@github.com:c-haos-onfusion/c-haos-onfusion.github.io.git
branch: gh-pages
    repo用ssh的，不然会报错10054

    输入三条命令
    hexo clean 
hexo g 
hexo d
    hexo d=hexo deploy

绑定域名

  • 在阿里云买域名
  • 配置域名
  • github配置
  • 在source文件夹下创建CNAME文件，写入购买的域名
  • 输入三条命令
    hexo clean 
hexo g 
hexo d

github创建免费图床

因为写博客需要插入图片，需要一个稳定图床保存图片，获取url，采用github的repo做图床，搭配picgo
  • 下载picgo
  • github新建图床repo 随便命名
  • 配置picgo

    其中token在github中用户settings/developer setting里

    至此，图床创建好了
    第一天还好好的，第二天发现图片全没了，显示不出来，开始查原因
    右键单击检查，发现错误是net::ERR_NAME_NOT_RESOLVED

    网上说是因为github ip经常变，需要在host文件中手动添加ip，具体步骤如下：
  • 查找raw.githubusercontent.com的ip
    这里使用这个网站查ip

    把这几个ip挨个ping，找最快的
  • 修改host文件
    位置在C:\Windows\System32\drivers\etc\hosts

    在最后一行添加ip和raw.githubusercontent.com

    再刷新下网站，图片就加载出来了

    还可以用switchhosts管理ip

修改主题

这里采用particlex的主题
  • 可按照particlex主题的教程修改
  • 修改 theme: hexo-theme-particlex-master

    注意！只修改这里只能在本地预览到主题的变更，但deploy后网络上主题没变
    需要修改url `url: https://c-haos-onfusion.github.io/`
  • 主题修改的其他细节参见所使用主题的教程
